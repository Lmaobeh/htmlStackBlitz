What is css

It stands for cascading style sheets standardizes how styles are applied to the web page.

In what order are the style rules are applied?

  1. Browser defaults
  2. Style tag information
  3. Inline style in tags

The scope gets smaller as you go down the list. However, each overrides eachother. So if define stlye information for all headers but want to override that I could add inline stlye tags

Style tags

basic syntax: h3 is called a selector because it selects the h3 tag to apply the style attribute to. 

      < style >
      	 h3 {
      	 	 Color: black;
      	 }
      < /style >

Selecting multiple tags: this makes H1 and h2 

      < style >
      	 h1, h2 {
      	 	 Color: blue;
      	 }
      < /style >

Defining css in a different file

if we have multiple sub pages, it becomes a hassle to copy the style information into every subpage html file. Duplication is never a good solution. .CSS files offer a central place to store the style information.
Style file: contains the contents of the style tags
Linking in the html file: rel attribute: relation type atrribute: text file with css, href atribute: link to the style file 

      < head &hgt
      	 < link rel="stylesheet" type="text/css" href="path to .css" >
